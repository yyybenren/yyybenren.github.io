摘要：本文介绍作者本人系统分析师考试论文学习方法，分析历年真题，集中复习常考方向，准备通用模板、知识点和范文，记忆重点内容，提高备考效率。

历年真题分析


观察近十年真题考察情况可以发现：

  1. 近几年论文主要集中在测试（图中黄色），开发模型(图中绿色)，开发方法（图中粉色）这三个章节
  2. 需求工程（图中蓝色）的内容2019年之前常考，近几年考察较少，猜测会进行考察
  3. 未上色为教程外的内容，主要涉及新技术及架构师内容较难把握出题方向
  4. 系统运维（灰色），系统规划分析（棕色）偶有考察

我的论文备考策略

从分析情况可以看出系统分析师论文考试出题方向是有所侧重的，除了2022年外历年都是提供四个题目选一作答，通过准备考察频率较高的内容进行复习基本能覆盖考试可能的出题角度

备考过程

  1. 准备通用的项目背景，摘要，结尾的模板
  2. 针对常考方向复习整理相关知识点
  3. 针对常考方向准备论文:将通用模版和题目要求内容进行组合
  4. 将自己编写的范文进行背诵记忆

重点记忆内容

  1. 测试（2）：系统测试（功能测试+性能测试）、自动化测试
  2. 需求方向（2）：需求获取、需求分析
  3. 系统开发方法（2）：结构化+面向对象、面向服务
  4. 软件开发模型（2）：UP模型、敏捷开发
  5. 系统运维（1）：遗产系统演化+数据迁移
